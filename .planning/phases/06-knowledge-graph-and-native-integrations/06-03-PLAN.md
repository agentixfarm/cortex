```
pub struct EntityStore {
    pub canonicals: HashMap<String, CanonicalEntity>,           // canonical_id -> entity
    pub alias_index: HashMap<(String, String), String>,         // (lowercase(value), entity_type) -> canonical_id
    pub doc_index: HashMap<String, std::collections::HashSet<String>>,  // canonical_id -> doc_ids
    pub canonical_embeddings: HashMap<String, Vec<f32>>,        // canonical_id -> embedding of canonical_name
}
```

Methods (all `pub` unless noted):
- `new() -> Self` — empty maps. Also impl Default.
- `rebuild_from_engine(&mut self, engine: &CortexEngine, embedder: &EmbeddingService) -> Result<(), AppError>` — O(N) scan of documents_384 collection per the edges.rs `build_edges` pattern (lines 47-87). Extract extracted_entities from each doc's metadata, for each entity call `find_or_create_canonical`, append doc_id to doc_index.
- `register_doc_entities(&mut self, doc_id: &str, entities: &[ExtractedEntity], embedder: &EmbeddingService) -> Result<Vec<String>, AppError>` — for each entity: lookup or create canonical via find_or_create_canonical (mutates entity.canonical_id? Return list of touched canonical_ids.
- `find_or_create_canonical(&mut self, surface: &str, entity_type: &str, embedder: &EmbeddingService) -> Result<String, AppError>` — private. Algorithm:
  1. Exact-match lookup in alias_index for (lowercase(surface), entity_type) → return existing.
  2. Embedding-based search: embed surface, cosine vs every canonical_embedding of same type. If best score ≥ 0.85: append surface to canonical.aliases, register in alias_index, return existing canonical_id.
  3. New canonical: uuid::Uuid::new_v4(), insert into canonicals/alias_index/canonical_embeddings.
- `run_full_alias_merge(&mut self, embedder: &EmbeddingService) -> Result<(), AppError>` — for each entity_type, pairwise compare canonicals (O(n²) within type), merge if cosine ≥ 0.85. Per RESEARCH §Anti-Patterns this runs ONCE after backfill, not per-doc.

Six commands per CONTEXT D-16 "Claude's Discretion" sketch (now finalized):

1. `get_entities_by_type(entity_type: Option<String>) -> Vec<EntitySummary>` — calls `store_guard.get_by_type(entity_type.as_deref())`.

2. `get_entity(id: String) -> CanonicalEntity` — calls `store_guard.get_canonical(&id).ok_or(AppError::NotFound(...))`.

3. `get_documents_for_entity(id: String) -> Vec<Document>` — read `store_guard.doc_index[&id]` to get doc_ids, drop the store lock, acquire engine.blocking_lock(), for each doc_id call `collection.db.get(doc_id)` and build the Document via the existing `crate::search::query::build_document_from_metadata` helper (used by get_document). Pattern mirrors documents.rs::get_related_documents lines 165-184.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Frontend IPC → entity commands | Untrusted args (entity_type strings, ids, alias strings, new_name strings) from renderer JS |
| Backfill task → engine | Concurrent reads/writes with the file watcher pipeline on the documents_384 collection |
